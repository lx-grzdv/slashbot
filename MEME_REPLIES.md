# Мемные реплики — документация

Случайные и принудительные мемы в @ag_slashbot на основе переписки в чате. Реализация: `meme_replies.py`, интеграция в `bot.py`.

Мемы **не заменяют** ответы в стиле Паши — это отдельный слой поверх обычной логики бота.

## Как это работает

```
Сообщение в чат
      │
      ▼
bot.py: handle_text_message()
      │
      ├── запись в историю чата (record_chat_message)
      │
      ├── @ag_slashbot / reply на бота? ──► ответ Паши (мема нет)
      ├── фоновый триггер (готово, макет…)? ──► ответ Паши (мема нет)
      │
      └── иначе ──► maybe_meme_reply()  (~3.5% в группе)
                    или /meme ──► force_meme_reply()
```

### История чата

Бот хранит в памяти **последние 24 сообщения** на каждый `chat_id`. История:

- накапливается на каждом входящем текстовом сообщении;
- **не пишется на диск** — сбрасывается при перезапуске;
- используется как контекст для LLM и шаблонов.

## Тон мемов

Мемы должны быть **кринжовыми и провокационными** — в духе реальных реплик Паши из S:P9, а не «корпоративным юмором».

| Ок | Не ок |
|---|---|
| `бля как же жопа горит от таких макетов` | `тишина в чате, макет на уровне гуд, ахах` |
| `понаехали дизайнеры` | `звучит как название стартапа` |
| `главный кринж дня: синк в 12` | `это же буквально мы` |
| `план говно: шапки магистерские` | мотивация, советы, нравоучения |

В коде:
- **`LLM_SYSTEM_PROMPT`** — задаёт тон для нейронки;
- **`BLAND_MEME`** — regex-фильтр, отсекает скучные шаблонные ответы LLM;
- **`MEME_TEMPLATES`** — кринжовые fallback-шаблоны из фраз чата.

## Случайные мемы

Срабатывают **только если** бот не ответил на сообщение обычным способом (нет `@`, нет reply на бота, нет фонового триггера Паши).

### Мем после тишины

Если в **группе** люди не писали **3 часа** (по умолчанию), бот кидает кринж-мем по контексту чата. Проверка каждые **20 минут**.

| Параметр | По умолчанию |
|---|---|
| `SILENCE_MEME_HOURS` | `3` |
| `SILENCE_MEME_CHECK_MIN` | `20` |
| `SILENCE_MEME_ENABLED` | `1` (выкл: `0`) |

- Срабатывает **один раз** за период тишины; после нового сообщения таймер сбрасывается.
- Только **group/supergroup**, не лички.
- Учитываются текстовые сообщения и команды от людей (не от ботов).

| Параметр | Группа | Личный чат |
|---|---|---|
| Шанс на сообщение | ~3.5% | ~1.2% |
| Кулдаун на чат | 7 мин (`MEME_COOLDOWN_SEC`) | то же |
| Минимум истории | 2 сообщения | 2 сообщения |
| Минимум длины текста | 6 символов | 6 символов |

**Контекст для генерации:**

- текущее сообщение;
- текст сообщения, на которое ответили (если есть reply);
- недавняя переписка из истории.

## Команда /meme

Принудительный мем без случайного шанса.

| Команда | Описание |
|---|---|
| `/meme` | Мем по контексту чата |
| `/мем` | То же (кириллица, через `handle_any_command`) |
| `/meme про макет` | Мем по тексту после команды |
| Reply + `/meme` | Мем с учётом сообщения, на которое ответили |

### Примеры

```
/meme
→ мем по последним сообщениям в чате

/meme синк и дедлайн горят
→ мем про эту фразу

[reply на «залил в фигму»] /meme
→ мем с опорой на то сообщение
```

### Ограничения команды

| Параметр | Значение |
|---|---|
| Кулдаун | **20 сек** на пару `(chat_id, user_id)` |
| LLM | всегда пробует нейронку первой (если есть ключ) |
| Пустой чат | fallback-промпт: «в чате тишина, все притворяются что макет гуд, а дедлайн горит» |

После `/meme` срабатывает и **общий кулдаун чата** (7 мин) — случайный мем сразу следом не придёт.

Команда зарегистрирована в меню Telegram: `/meme — Принудительный мем`.

## Генерация: LLM и шаблоны

### С OPENAI_API_KEY

1. Запрос в OpenAI Chat Completions (`gpt-4o-mini` по умолчанию).
2. Промпт заточен под **кринж и провокацию** в стиле Паши (мат, зашквар, пассивная агрессия) — не bland-корпоративный юмор.
3. Скучные ответы («тишина в чате», «на уровне гуд»…) отбрасываются фильтром `BLAND_MEME`.
4. `/meme` — до 3 попыток LLM; случайный мем — до 2.
5. При ошибке API или невалидном ответе — **fallback на шаблоны** (тоже кринжовые).

### Без OPENAI_API_KEY

Только **шаблоны** — целые фразы из чата (не обрывки слов), иногда микс из двух реплик:

```
бля: надо делать инфо по промо. и тут же Спасибо, что не с пивом. классика
сначала Катя оставит коммент во фрейме, потом люди тычут истошно — ну всё, дудосинг
```

Обрывки вроде «че макетам» отфильтровываются (`BROKEN_FRAGMENT`, `_is_coherent_snippet`).

### Когда какой режим

| Сценарий | LLM | Шаблоны |
|---|---|---|
| Случайный мем, ключ есть | ~85% (`MEME_LLM_CHANCE`) | ~15% + fallback |
| Случайный мем, ключа нет | — | 100% |
| `/meme` / `/мем`, ключ есть | всегда сначала LLM | fallback при ошибке |
| `/meme` / `/мем`, ключа нет | — | 100% |

Зависимости для LLM **не нужны** — используется `urllib` из стандартной библиотеки.

## Переменные окружения

Задаются в Railway → **Variables** или локально в `.env` / `config.py`.

| Переменная | Обязательно | По умолчанию | Назначение |
|---|---|---|---|
| `OPENAI_API_KEY` | нет | — | Включает LLM-генерацию мемов |
| `MEME_LLM_MODEL` | нет | `gpt-4o-mini` | Модель OpenAI |
| `MEME_LLM_CHANCE` | нет | `0.85` | Доля случайных мемов через LLM (0–1) |
| `MEME_LLM_TIMEOUT_SEC` | нет | `12` | Таймаут запроса к API, сек |
| `OPENAI_BASE_URL` | нет | `https://api.openai.com/v1` | Базовый URL (прокси / совместимый API) |
| `SILENCE_MEME_HOURS` | нет | `3` | Часов тишины в группе до мема |
| `SILENCE_MEME_CHECK_MIN` | нет | `20` | Как часто проверять тишину, мин |
| `SILENCE_MEME_ENABLED` | нет | `1` | `0` — выключить мем после тишины |

`OPENAI_BASE_URL` не нужен при обычном ключе OpenAI.

Пример для Railway:

```
OPENAI_API_KEY=sk-...
MEME_LLM_MODEL=gpt-4o-mini
MEME_LLM_CHANCE=0.85
```

## Настройка в коде

Константы в `meme_replies.py`:

| Константа | По умолчанию | Описание |
|---|---|---|
| `MEME_CHANCE_GROUP` | `0.035` | Шанс случайного мема в группе |
| `MEME_CHANCE_PRIVATE` | `0.012` | Шанс в личке |
| `MEME_COOLDOWN_SEC` | `420` | Кулдаун случайных мемов на чат (7 мин) |
| `MEME_FORCE_COOLDOWN_SEC` | `20` | Кулдаун `/meme` на пользователя |
| `MEME_HISTORY_SIZE` | `24` | Размер буфера истории |
| `MEME_MIN_HISTORY` | `2` | Минимум сообщений для случайного мема |
| `MEME_MIN_TEXT_LEN` | `6` | Минимальная длина текста |
| `MEME_MAX_LEN` | `140` | Максимальная длина мема |
| `BLAND_MEME` | regex | Отсекает bland-ответы LLM («тишина в чате», «на уровне»…) |

Шаблоны — список `MEME_TEMPLATES`, промпт LLM — `LLM_SYSTEM_PROMPT` в том же файле.

## Логи и отладка

В логах Railway / локального запуска:

| Строка | Значение |
|---|---|
| `🎭 Мемная реплика в чат …` | Случайный мем отправлен |
| `🎭 Force meme в чат …` | Мем по команде `/meme` |
| `🧠 LLM meme: …` | Ответ сгенерировала нейронка |
| `⚠️ LLM meme failed: …` | Ошибка API → fallback или пустой ответ |

### Быстрая проверка локально

```bash
python3 -c "
import asyncio
import meme_replies as mr

async def main():
    chat_id, user_id = 1, 1
    for t in ['залил макет', 'синк в 12', 'шапки не влезают']:
        mr.record_chat_message(chat_id, t)
    meme, err = await mr.force_meme_reply(chat_id, user_id, prompt_text='про дедлайн')
    print(meme or err)

asyncio.run(main())
"
```

### Типичные ситуации

| Симптом | Причина |
|---|---|
| Мемы не появляются сами | Нормально: шанс ~3.5%, кулдаун 7 мин |
| `/meme` → «подожди 20 сек» | Кулдаун команды на пользователя |
| `/meme` → «не смог сгенерить» | Пустой контекст и не сработали шаблоны |
| В логах `LLM meme failed` | Неверный ключ, лимит API, таймаут — проверь Variables |
| После рестарта мемы «не в тему» | История в памяти обнулилась — нужно несколько сообщений в чате |

## API модуля

```python
from meme_replies import (
    record_chat_message,
    maybe_meme_reply,
    force_meme_reply,
)

# На каждое входящее сообщение (уже в bot.py)
record_chat_message(chat_id, message_text)

# Случайный мем (async)
meme = await maybe_meme_reply(
    chat_id,
    message_text,
    chat_type="group",
    reply_to_text=optional_reply_text,
)

# Принудительный мем (async)
meme, error = await force_meme_reply(
    chat_id,
    user_id,
    prompt_text="про макет",      # опционально
    reply_to_text=optional_text,  # опционально
)
```
