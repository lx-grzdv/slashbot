# Мемные реплики — документация

Случайные и принудительные мемы в @ag_slashbot на основе переписки в чате. Реализация: `meme_replies.py`, интеграция в `bot.py`.

Мемы **не заменяют** ответы в стиле Паши — это отдельный слой поверх обычной логики бота.

## Как это работает

```
Сообщение в чат
      │
      ▼
bot.py: handle_text_message()
      │
      ├── запись в историю чата (record_chat_message)
      │
      ├── @ag_slashbot / reply на бота? ──► ответ Паши (мема нет)
      ├── фоновый триггер (готово, макет…)? ──► ответ Паши (мема нет)
      │
      └── иначе ──► maybe_meme_reply()  (~3.5% в группе)
                    или /meme ──► force_meme_reply()
```

### История чата

Бот хранит два слоя контекста на каждый `chat_id`:

- **последние 24 сообщения** — короткий буфер для быстрых случайных мемов;
- **сообщения текущего дня по МСК** — до 220 строк, основной контекст для `/meme` и плановых мемов S:P9;
- оба буфера накапливаются на каждом входящем текстовом сообщении;
- оба буфера **сохраняются на диск** в `meme_state.json` (каталог `SLASHBOT_DATA_DIR`, на Railway — `/data`);
- дневной контекст передаётся LLM с простым списком «тем дня», чтобы мем цеплялся за реальные события дня;
- активность чата (для «мема после тишины») тоже персистится.

Без Volume на Railway история **сбрасывается при redeploy**. См. [OPERATIONS.md](OPERATIONS.md).

## Тон мемов

Мемы должны быть **острыми, самоироничными и поддерживающими**. Бот шутит над рабочим хаосом, фигмой и процессом, но не обесценивает людей и их работу. Финал даёт живой импульс — «дожмём», «вывезем», «погнали» — без корпоративной мотивации.

| Ок | Не ок |
|---|---|
| `блэт, фигма опять проверяет нас на прочность — сейчас дожмём` | `прогресса ноль, все притворяются` |
| `макет сделал крюк, команда сделала выводы — погнали` | `макеты висят, рендер мёртв` |
| `собираем хвосты без геройства` | `звучит как название стартапа` |
| `что успели — молодцы, остальное переживёт выходные` | корпоративные советы и нравоучения |

В коде:
- **`LLM_SYSTEM_PROMPT`** — задаёт тон для нейронки;
- **`BLAND_MEME`** — regex-фильтр, отсекает скучные шаблонные ответы LLM;
- **`DISCOURAGING_SCHEDULED_MEME`** — блокирует безысходные плановые реплики;
- **`ENCOURAGING_SCHEDULED_MEME`** — требует в плановом меме маркер движения, поддержки или здорового отдыха;
- **`MEME_TEMPLATES`** — кринжовые fallback-шаблоны из фраз чата.

## Случайные мемы

Срабатывают **только если** бот не ответил на сообщение обычным способом (нет `@`, нет reply на бота, нет фонового триггера Паши).

### Мем после тишины

Если в **группе** люди не писали **3 часа** (по умолчанию), бот кидает кринж-мем по контексту чата. Проверка каждые **20 минут**.

| Параметр | По умолчанию |
|---|---|
| `SILENCE_MEME_HOURS` | `3` |
| `SILENCE_MEME_CHECK_MIN` | `20` |
| `SILENCE_MEME_ENABLED` | `1` (выкл: `0`) |

- Срабатывает **один раз** за период тишины; после нового сообщения таймер сбрасывается.
- Только **group/supergroup**, не лички.
- Учитываются текстовые сообщения и команды от людей (не от ботов).

### Плановые мемы в S:P9 works

Отдельно от случайных мемов — **по расписанию** в чат `-1002413642408` (см. [SP9_WORKS_CHAT.md](SP9_WORKS_CHAT.md)):

| Время (МСК, ПН–ПТ) | Слот | Содержание |
|---|---|---|
| **15:00** | `afternoon` | Послеобеденняя поддерживающая ирония и импульс к движению |
| **18:00** | `evening` | Дожать главное без геройства и закончить день |
| **18:00 пятница** | `evening_friday` | Зафиксировать победы недели и отпустить остальное без вины |

| Параметр | По умолчанию |
|---|---|
| `SP9_SCHEDULED_MEME_ENABLED` | `1` (выкл: `0`) |
| `SP9_AFTERNOON_MEME_HOUR` / `MIN` | `15` / `0` |
| `SP9_EVENING_MEME_HOUR` / `MIN` | `18` / `0` |

Генерация: LLM по **сегодняшней переписке** + явный фокус слота (послеобед / вечер / пятница) + fallback-шаблоны. Если дневной буфер пустой, используется короткая последняя история. Если и она пуста, бот выбирает только безопасный готовый fallback; `focus` остаётся служебной инструкцией и никогда не передаётся в шаблоны как текст чата. История сохраняется в `meme_state.json` (см. ниже). В логах: `🌤️ SP9 scheduled meme (afternoon)…`, `🌆 … (evening)…`, `🍻 … (evening_friday)…`.

Для плановых мемов действует дополнительная гарантия тона: `_is_inspiring_scheduled_meme()` блокирует безысходный финал и требует маркер движения, поддержки или здорового отдыха. Неподходящий ответ LLM запускает повторную попытку; после исчерпания попыток используется поддерживающий fallback.

| Параметр | Группа | Личный чат |
|---|---|---|
| Шанс на сообщение | ~3.5% | ~1.2% |
| Кулдаун на чат | 7 мин (`MEME_COOLDOWN_SEC`) | то же |
| Минимум истории | 2 сообщения | 2 сообщения |
| Минимум длины текста | 6 символов | 6 символов |

**Контекст для генерации:**

- текущее сообщение;
- текст сообщения, на которое ответили (если есть reply);
- сегодняшняя переписка из дневного буфера;
- темы дня, если сообщений достаточно много.

## Команда /meme

Принудительный мем без случайного шанса.

| Команда | Описание |
|---|---|
| `/meme` | Мем по контексту чата |
| `/мем` | То же (кириллица, через `handle_any_command`) |
| `/meme про макет` | Мем по тексту после команды |
| Reply + `/meme` | Мем с учётом сообщения, на которое ответили |

### Примеры

```
/meme
→ мем по последним сообщениям в чате

/meme синк и дедлайн горят
→ мем про эту фразу

[reply на «залил в фигму»] /meme
→ мем с опорой на то сообщение
```

### Ограничения команды

| Параметр | Значение |
|---|---|
| Кулдаун | **20 сек** на пару `(chat_id, user_id)` |
| LLM | всегда пробует нейронку первой (если есть ключ) |
| Пустой чат | fallback-промпт: «в чате тишина, все притворяются что макет гуд, а дедлайн горит» |

После `/meme` срабатывает и **общий кулдаун чата** (7 мин) — случайный мем сразу следом не придёт.

Команда зарегистрирована в меню Telegram: `/meme — Принудительный мем`.

## Генерация: LLM и шаблоны

### С OPENAI_API_KEY

1. Запрос в OpenAI Chat Completions (`gpt-4o-mini` по умолчанию).
2. Промпт случайно выбирает один из трёх режимов: кринж S:P9, «дур-дачник» или пародийный «силовик без глянца».
3. Скучные ответы («тишина в чате», «на уровне гуд»…) отбрасываются фильтром `BLAND_MEME`.
4. `/meme` — до 3 попыток LLM; случайный мем — до 2.
5. При ошибке API или невалидном ответе — **fallback на шаблоны** (тоже кринжовые).

### Без OPENAI_API_KEY

Только **шаблоны** — целые фразы из чата (не обрывки слов), иногда микс из двух реплик:

```
бля: надо делать инфо по промо. и тут же Спасибо, что не с пивом. классика
сначала Катя оставит коммент во фрейме, потом люди тычут истошно — ну всё, дудосинг
```

Обрывки вроде «че макетам» отфильтровываются (`BROKEN_FRAGMENT`, `_is_coherent_snippet`).

### Когда какой режим

| Сценарий | LLM | Шаблоны |
|---|---|---|
| Случайный мем, ключ есть | ~85% (`MEME_LLM_CHANCE`) | ~15% + fallback |
| Случайный мем, ключа нет | — | 100% |
| `/meme` / `/мем`, ключ есть | всегда сначала LLM | fallback при ошибке |
| `/meme` / `/мем`, ключа нет | — | 100% |

Зависимости для LLM **не нужны** — используется `urllib` из стандартной библиотеки.

| `BLAND_MEME` | regex | Отсекает bland-ответы LLM («тишина в чате», «на уровне»…) |
| `PROMPT_LEAK` | regex | Блокирует служебные маркеры `focus`/промпта до отправки в Telegram |
| `DISCOURAGING_SCHEDULED_MEME` | regex | Отсекает безысходные плановые мемы («прогресса ноль», «макеты висят», «мёртв») |
| `ENCOURAGING_SCHEDULED_MEME` | regex | Требует маркер движения, поддержки или здорового отдыха |

Шаблоны — списки `MEME_TEMPLATES`, `MASHUP_TEMPLATES`, `DURDACH_FALLBACKS`, `DURDACH_MASHUP_TEMPLATES`, `SMAEV_FALLBACKS`, `SMAEV_MASHUP_TEMPLATES`; промпт LLM — `LLM_SYSTEM_PROMPT` в том же файле.

### Режим «дур-дачник»

Плановые мемы S:P9 иногда переключаются в отдельный дачно-речной стиль: рабочий бред объясняется через реку, сапоги, тину, термос, чай, пень, ведро, камыш, комаров и бесклёвье. Это не длинный монолог про «преисполнился», а короткая реплика сельского чудика с мягкой поддёвкой.

Примеры направления:

```
мне бы к камышам, а не к этому checkout — сапог в тине и то понятнее держится
коммент во фрейме лежит как ведро без ручки: вроде вещь, а нести стыдно
синк ваш нервный, как карась перед грозой; я бы чай налил и не трогал макет до утра
```

### Режим «силовик без глянца»

Ещё одна случайная ветка плановых мемов — безопасный пародийный архетип по мотивам исследования речевого образа Андрея Смаева. Он говорит коротко и конкретно, переводит рабочий контекст в нагрузку, подходы, режим, технику и результат. Бот не пишет от имени Смаева, не использует реальные цитаты и биографические детали.

Примеры направления:

```
Макет тяжёлый. Подход рабочий. Дожимаем без геройства.
Режим сбит. Фигма грузится. Работаем из того, что есть.
Комментов много. Нагрузка хорошая. Технику держим, результат будет.
```

## Логи и отладка

В логах Railway / локального запуска:

| Строка | Значение |
|---|---|
| `🎭 Мемная реплика в чат …` | Случайный мем отправлен |
| `🎭 Force meme в чат …` | Мем по команде `/meme` |
| `🌤️ SP9 scheduled meme (afternoon)…` | Плановый мем 15:00 МСК |
| `🌆 SP9 scheduled meme (evening)…` | Плановый мем 18:00 МСК |
| `🍻 SP9 scheduled meme (evening_friday)…` | Пятничное напутствие 18:00 |
| `🧠 LLM meme: …` / `🧠 LLM scheduled meme: …` | Ответ сгенерировала нейронка |
| `✅ LLM: …` (при старте) | Проверка API прошла |
| `⚠️ LLM meme failed: …` | Ошибка API → fallback |
| `💾 История чатов загружена: …` | `meme_state.json` прочитан с диска |

### Быстрая проверка локально

```bash
python3 -c "
import asyncio
import meme_replies as mr

async def main():
    chat_id, user_id = 1, 1
    for t in ['залил макет', 'синк в 12', 'шапки не влезают']:
        mr.record_chat_message(chat_id, t)
    meme, err = await mr.force_meme_reply(chat_id, user_id, prompt_text='про дедлайн')
    print(meme or err)

asyncio.run(main())
"
```

### Типичные ситуации

| Симптом | Причина |
|---|---|
| Мемы не появляются сами | Нормально: шанс ~3.5%, кулдаун 7 мин |
| `/meme` → «подожди 20 сек» | Кулдаун команды на пользователя |
| `/meme` → «не смог сгенерить» | Пустой контекст и не сработали шаблоны |
| В логах `LLM meme failed` | Неверный ключ, лимит API, таймаут — проверь Variables |
| После рестарта мемы «не в тему» | История в памяти обнулилась — нужно несколько сообщений в чате |
| В меме видны `ПОСЛЕОБЕДЕННЫЙ мем`, `Стиль можно как` или `РЕЖИМ ДУР-ДАЧНИК` | Утечка служебного `focus`; проверь, что в проде есть коммит `cb300a3`, и перезапусти сервис |

## Инцидент 2026-07-13: утечка focus-промпта

### Симптомы

Плановые мемы в 15:00 и 18:00 МСК опубликовали части внутренней инструкции генератора, обёрнутые в обычные fallback-шаблоны. Характерные маркеры: `ПОСЛЕОБЕДЕННЫЙ мем (15:00 МСК)`, `Стиль можно как кринж S:P9`, `Дур-Дачник`.

### Первопричина

1. Дневная и короткая история чата были пусты.
2. LLM была недоступна, вернула ошибку или невалидный ответ.
3. `_generate_scheduled_sp9_meme()` подставляла `focus` в источники фраз: `sources = [focus]`.
4. `_build_meme()` считала служебную инструкцию репликой чата и вставляла её в `{snippet}`.

### Исправление

- Коммит: `cb300a3 Prevent scheduled meme prompt leaks`, запушен в `origin/main` 2026-07-13.
- При пустой истории `_generate_scheduled_sp9_meme()` сразу выбирает готовый безопасный fallback.
- `focus` используется только в LLM-запросе.
- `PROMPT_LEAK` добавлен в `_is_valid_meme()` и отсекает маркеры промпта как в fallback, так и в ответе LLM.

### Проверка

- `test_meme_replies.py` содержит регрессионные тесты на оба текста из инцидента, пустую историю и обычную историю.
- Перед деплоем: 3/3 теста прошли, синтаксис проверен, 1500 генераций fallback для трёх слотов прошли без утечек.
- После push исправления в `origin/main` в `S:P9 works` вручную отправлен проверочный мем, Telegram `message_id=20111`: «блэт, бот дважды зачитал техзадание вслух — даже фигма постеснялась так палить внутреннюю кухню». Это подтверждает ручную отправку и валидацию текста, но не статус Railway deployment.

### Если утечка повторится

1. Временно установить `SP9_SCHEDULED_MEME_ENABLED=0` и сделать redeploy.
2. Проверить в Railway, что развёрнут `cb300a3` или более новый `main`.
3. Проверить логи `LLM meme failed` и загрузку `meme_state.json`.
4. Добавить точный утёкший текст в `test_rejects_prompt_leaks_from_screenshots` и расширить `PROMPT_LEAK`, если появился новый маркер.

## API модуля

```python
from meme_replies import (
    record_chat_message,
    maybe_meme_reply,
    force_meme_reply,
)
```
