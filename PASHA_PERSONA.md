## Идея и ограничения

### Что бот делает

- **Реагирует** на сообщения короткими фразами в манере Паши.
- **Синтезирует** новые реплики из блоков слов (не только копирует цитаты).
- **Учитывает контекст** входящего сообщения (письмо, макет, синк и т.д.).

### Чего бот не делает (намеренно)

Бот **не должен** звучать как менеджер, который что-то сделал или собирается сделать.

| Запрещённый тип | Примеры |
|---|---|
| «Я сделал» | оставил комменты, откомментил, закинул |
| «Сделай ты» | проверь плиз, дай ссылку, закинь на диск |
| «Сделаю» | завтра гляну, дам сигнал, подвинем |
| «Сейчас приду» | бегу, 5 мин, мин (для синка/захода) |

Фильтр реализован в константе `FORBIDDEN` в `pasha_persona.py`. Каждая сгенерированная фраза проходит проверку перед отправкой.

### Допустимые типы реакций

| Тип | Примеры |
|---|---|
| Согласие | да, ага, угу, дада |
| Оценка | гуд, норм, каеф, збс, стопроц |
| Благодарность | спасибо, спс, супер спасибо, граци рагаци |
| Эмоция | блэт, лол, ахах, ура, да бля |
| Задумчивость | хз, ну, вроде, как скажете, чет хз |
| Социальное | салам, парни, красавчики |

## Архитектура

```
Telegram
   │
   ▼
bot.py                          pasha_persona.py
├── CommandHandler /pasha  ──►  generate_pasha_response()
├── handle_any_command()   ──►  generate_pasha_response(command=...)
└── handle_text_message()  ──►  pasha_reply_to_message()
                                synthesize_reaction()
                                     │
                                     ├── _detect_context()     ← триггеры в тексте
                                     ├── _synthesize_one()       ← сборка фразы
                                     ├── _synthesize_thoughtful()← синк/заход/го
                                     └── FORBIDDEN + длина      ← валидация
```

### Файлы

| Файл | Назначение |
|---|---|
| `pasha_persona.py` | Словари, триггеры, синтез, публичный API |
| `bot.py` | Telegram-хендлеры, упоминания, reply на бота |
| `PASHA_PERSONA.md` | Эта документация |

Остальной функционал slashbot (расписание, рассылки, Whereby, веб-панель) **не затронут** персоной Паши.

## Когда бот отвечает

### Всегда отвечает

1. **Упоминание** `@ag_slashbot` в тексте.
2. **Reply** на сообщение бота (по username `ag_slashbot`).
3. **Любая slash-команда**, кроме служебных (см. [Команды](#команды-telegram)).

### Отвечает без упоминания (фоновый режим)

Если в сообщении есть **сильный рабочий триггер** (`BACKGROUND_TRIGGER`):

```
готово, сделал, залил, закинул, отдал, загрузил, обновил, поправил,
заход, письмо, email, промо, макет, figma, баннер, ленд,
видео, рилс, апрув, ревью, синк, созвон, whereby
```

### S:P9 works — шире фоновых триггеров

В чате **S:P9 works** (`chat_id -1002413642408`) бот дополнительно реагирует на любой **рабочий контекст** из `TRIGGER_CONTEXT`: приветствия, спасибо, баги, «го?», переводы и т.д. — без `@ag_slashbot`. В остальных чатах — только `BACKGROUND_TRIGGER` и упоминания.

См. [SP9_WORKS_CHAT.md](SP9_WORKS_CHAT.md).

### Не отвечает

- Обычный чат без упоминания и без триггеров.
- Служебные команды (`/set_schedule`, `/help`, …).
- Сообщения, для которых синтез не прошёл валидацию (редко — тогда fallback на якорную фразу).

### Особый случай: «Заход»

Сохранено поведение оригинального slashbot:

- При слове **«Заход»** / **«заход»** бот **через 60 секунд** отправляет: `Заход на завод`.
- Параллельно может дать **задумчивую** пашину реакцию (если сработал `BACKGROUND_TRIGGER`).

## Триггерные слова

Триггеры сопоставляются regex-ом в `TRIGGER_CONTEXT`. Первое совпадение определяет **контекст** ответа.

### Таблица триггеров

| Контекст | Ключевые слова в сообщении | Пример входа |
|---|---|---|
| `delivered` | готово, сделал, залил, закинул, отдал, загрузил, обновил, поправил, отправил | «залил макет на диск» |
| `mail` | письм, email, промо, рассыл | «письмо готово» |
| `design` | макет, фигм, figma, верст, баннер, ленд, обложк, квиз | «глянь баннер в figma» |
| `video` | видео, рилс, ролик, монтаж, ftmo | «рилс залили» |
| `task` | задач, notion, ноушн, тикет | «задача в ноушне» |
| `approve` | апрув, approve, ревью, review, гуд, норм, ок | «на апрув» |
| `sync` | синк, созвон, мит, meet, whereby, колл, **заход** | «синкуемся?» |
| `translate` | перевод, локал | «переводы готовы» |
| `greeting` | привет, салам, здаров, доброе утро | «привет парни» |
| `thanks` | спасибо, спс, благодар | «спасибо, глянул» |
| `problem` | ошиб, баг, не работ, сломал, проблем, не могу | «тут баг в верстке» |
| `go` | го, го? | «го?» |

Если ни один триггер не сработал — контекст `generic`.

## Контексты и типы реакций

### Общий режим (большинство контекстов)

Короткие **оценки и согласия**, без обещаний.

| Контекст | Типичные ответы |
|---|---|
| `delivered` | Спасибо, да, Гуд, Ага, супер спасибо |
| `mail` | да, Спасибо, Ага, гуд |
| `design` | Да, Спасибо, ага, норм |
| `video` | блэт, каеф, лол, гуд |
| `task` | Да, Спасибо, ага |
| `approve` | стопроц, апрувед, гуд, каеф |
| `translate` | Да, Ага, Спасибо |
| `greeting` | салам, го?, парни, ага |
| `thanks` | граци рагаци, красавчики, мазлтов! |
| `problem` | да бля, блэт, лол, ого |
| `generic` | случайная якорная или синтезированная реакция |

### Приоритет контекста

Если в одном сообщении несколько триггеров, используется **первый** по порядку в `TRIGGER_CONTEXT` (сверху вниз в таблице выше).

> **Важно:** `problem` стоит **выше** `design`, чтобы «баг в верстке» попадал в problem, а не в design из‑за слова «верст».

## Провокационные и длинные реакции

Помимо коротких «ага / спасибо», бот умеет отвечать **колко и иронично** — как Паша в S:P9 works.

### Короткие провокационные блоки

| Константа | Примеры |
|---|---|
| `PROVOCATIVE` | блэт, да бля, договор, нuu неее, все серьезно, не, так хуже |
| `PROVOCATIVE_TEASE` | красавчики, мазлтov!, граци рагаци, секси |
| `PROVOCATIVE_COMPOUND` | классика, блэт / лавли, спасибище / да. но херово |
| `EMPHASIS` | да бля, ++++, lf, Неа |

Чаще включаются в контекстах `problem`, `video`, `approve`, `thanks`, иногда `generic`.

### Длинные контекстные цитаты

| Константа | Описание |
|---|---|
| `PROVOCATIVE_LONG` | ~70 реальных фраз из экспорта (33–120 символов), разбиты по контекстам |
| `DELIVERED_ANCHORS` | только «спасибо / гуд / ага» — без провокации на «готово» |

**Примеры длинных:**

| Контекст | Фраза |
|---|---|
| `problem` | «чУприн блядь. Пonaехали дизайнеры» |
| `design` | «бля как же жопа горит от таких макетов» |
| `delivered` | «хихи какой прикольный первый слайд получился) мне все нравится, забираю, спасибо!» |
| `generic` | «Минская классика - хуй где сядешь» |

Имена коллег (Саня, Алексис, Леха) **сохранены** — это часть стиля.

### Вероятность длинного ответа

| Константа | Значение |
|---|---|
| `MAX_SHORT_LEN` | 32 |
| `MAX_LONG_LEN` | 120 |
| `LONG_REPLY_CHANCE` | 0.28 (базовая) |

Фактическая доля длинных ответов выше на `problem` / `design` (~38%) и ниже на `delivered` / `sync` (~12–18%). На «сдали работу» из generic-пула длинные **не** подмешиваются — только позитивные из `delivered`.

## Синтез реакций

Бот не только выбирает готовую фразу, но и **собирает новые** из блоков.

### Блоки слов

| Блок | Константа | Примеры |
|---|---|---|
| Согласие | `AFFIRM`, `AFFIRM_SIMPLE` | да, ага, го, угу, неа |
| Оценка | `EVAL`, `EVAL_SIMPLE` | гуд, норм, каеф, збс, стопроц |
| Спасибо | `THANKS` | спасибо, спс, супер спасибо |
| Вводные | `LEAD` | ну, кста, чет, вроде, ого |
| Усиление | `EMPHASIS` | да бля, ++++, lf |
| Социальное | `SOCIAL` | парни, салам, красавчики |
| Готовые пары | `COMPOUND` | супер спасибо, ну да, граци рагаци |
| Якоря | `ANCHORS` | реальные частые реплики из чата |
| Провокация | `PROVOCATIVE`, `PROVOCATIVE_TEASE` | блэт, договор, красавчики |
| Длинные цитаты | `PROVOCATIVE_LONG` | фразы 33–120 символов по контекстам |
| Сдача работы | `DELIVERED_ANCHORS` | спасибо, гуд, ага (без колкости) |

### Алгоритм `synthesize_reaction(text)`

```
1. Определить контекст → _detect_context(text)
2. С вероятностью _long_reply_chance(context):
   a. Выбрать фразу → _pick_long_reaction(context)
   b. Принять, если длина ≤ 120 и прошла FORBIDDEN / URL
3. Иначе до 10 коротких попыток:
   a. Собрать фразу → _synthesize_one(context)
   b. Отбросить, если длина > 32 символов
   c. Отбросить, если совпало FORBIDDEN
   d. Отбросить, если есть URL/@
4. Fallback → случайный ANCHORS
```

### Шаблоны склейки

| Шаблон | Пример результата |
|---|---|
| якорь (35% шанс) | `каеф` |
| готовая пара | `супер, спасибо` |
| `{lead}, {eval}` | `кста, збс` |
| `{lead} {affirm}` | `ого ага` |
| `{affirm} {eval}` | `да стопроц` |
| `{eval}!` | `каеф!` |

### Регистр

Функция `_maybe_cap()` с вероятностью ~45% делает заглавной первую букву односложных слов — как у Паши: `Да`, `Ага`, `Спасибо`.

## Задумчивый режим (синк / заход)

Для контекстов **`sync`** и **`go`** используется отдельный синтезатор `_synthesize_thoughtful()`.

### Почему отдельно

Фразы вроде `го`, `5 мин`, `бегу` на синк/заход звучат как **обещание прийти**. Бот вместо этого отвечает **в раздумьях**.

### Словарь задумчивых реакций

**Одиночные:** `хз`, `ну`, `вроде`, `наверн`, `чет`, `как скажете`, `lf`, `ого`, `неа`, `интересно`, `странно`, `сложно`

**Составные:** `ну да`, `ну такое`, `да хз`, `чет хз`, `вроде да`, `не уверен`, `ниче непонятно`

### Шаблоны задумчивого синтеза

| Шаблон | Пример |
|---|---|
| случайная из `THOUGHTFUL` | `хз` |
| случайная из `THOUGHTFUL_COMPOUND` | `как скажете` |
| `{lead} {tail}` | `чет хз` |
| `вроде {tail}` | `вроде норм` |
| `чет {…}` | `чет странно` |
| `ну, {…}` | `ну, сложно` |

### Примеры

| Вход | Возможный ответ |
|---|---|
| `синкуемся?` | `как скажете` |
| `заход` | `чет хз` |
| `созвон в 12` | `ну, интересно` |
| `го?` | `вроде да` |
| `/синк` | `ну такое` |

## API модуля

### `generate_pasha_response(text=None, command=None) -> str`

Главная точка входа. Всегда возвращает строку.

```python
from pasha_persona import generate_pasha_response

generate_pasha_response(text="залил макет")
# → "Спасибо" / "гуд" / "супер, спасибо" / …

generate_pasha_response(command="синк")
# → "чет хз" / "как скажете" / …
```

### `pasha_reply_to_message(message_text) -> str | None`

Решает, нужно ли отвечать на сообщение **без явного вызова команды**.

- `@ag_slashbot …` → всегда ответ.
- Сильный триггер без упоминания → ответ.
- Иначе → `None` (молчит).

### `synthesize_reaction(text=None) -> str`

Низкоуровневый синтез с валидацией.

### `sample_synthetic(count=20, context=None) -> list[str]`

Генерация пачки примеров для отладки.

```python
from pasha_persona import sample_synthetic

sample_synthetic(10, "sync")    # 10 задумчивых
sample_synthetic(10, "video") # 10 для видео-контекста
```

### Константы

| Константа | Значение |
|---|---|
| `BOT_USERNAME` | `ag_slashbot` |
| `BOT_MENTION` | `@ag_slashbot` |
| `MAX_SHORT_LEN` | `32` |
| `MAX_LONG_LEN` | `120` |
| `LONG_REPLY_CHANCE` | `0.28` |

## Настройка и запуск

### Требования

- Python 3.9+
- Зависимости из `requirements.txt`
- Токен бота в `config.py` или `BOT_TOKEN` в env

### Локальный запуск

```bash
cd slashbot
pip install -r requirements.txt
export BOT_TOKEN="..."
python3 bot.py
```

### Telegram: Privacy Mode

Чтобы бот видел сообщения в группе без упоминания (фоновые триггеры):

1. @BotFather → Bot Settings → Group Privacy → **Turn off**, или
2. Назначить бота **администратором** группы.

Без этого в группах работают только `@ag_slashbot`, reply и slash-команды.

### Деплой

См. [DEPLOY.md](DEPLOY.md) и [RAILWAY_SETUP.md](RAILWAY_SETUP.md).

- В репозитории должны быть `pasha_persona.py` и `bot.py` рядом.
- `/pasha` — `CommandHandler`; `/паша` — только латиница в API Telegram, кириллица обрабатывается в `handle_any_command` (`bot.py`).
- На Railway: `Procfile` → `start_both.py` (бот в главном потоке, веб в фоне).

## Как расширять

### Добавить новый триггер

В `pasha_persona.py`:

1. Добавить regex в `TRIGGER_CONTEXT`:
   ```python
   (re.compile(r"лента|stories", re.I), "design"),
   ```
2. При необходимости — веса в `CONTEXT_WEIGHTS`.
3. Опционально — контекстные шаблоны в `_synthesize_one()`.

### Добавить новые реакции

- В **якоря** → `ANCHORS` (реальные фразы из чата).
- В **блоки** → `EVAL`, `THANKS`, `THOUGHTFUL`, `PROVOCATIVE` и т.д.
- В **длинные цитаты** → `PROVOCATIVE_LONG[context]` (33–120 символов, без `@`, без FORBIDDEN).
- **Не добавлять** фразы с глаголами действия — их отсечёт `FORBIDDEN`, но лучше не засорять пулы.

### Добавить запрет

Расширить regex `FORBIDDEN`:

```python
FORBIDDEN = re.compile(
    r"...|новый_запрет",
    re.I,
)
```

### Изменить задумчивый режим

Редактировать `THOUGHTFUL`, `THOUGHTFUL_COMPOUND` и `_synthesize_thoughtful()`.

2. Фильтрация: длина ≤ 28–55 символов, без URL, без глаголов действия.
3. Частотный анализ слов, биграмм, стартеров фраз.
4. Сопоставление «сообщение коллеги → следующая реплика Паши» для триггеров.
5. Выделение задумчивых реплик отдельно от обещаний по времени.

### Ключевые находки анализа

- Самые частые слова Паши: **можно**, **плиз**, **спасибо**, **комменты** — но для бота комменты/плиз отфильтрованы как обещания/действия.
- Самые безопасные реакции: **да**, **ага**, **спасибо**, **гуд**, **го**, **каеф**.
- На **синк/заход** в данных часто шли «го», «5 мин» — для бота заменены на **задумчивый** режим по требованию.
