# S:P9 works — чат бота

Зафиксированный чат:

- `title`: `S:P9 works`
- `chat_id`: `-1002413642408` (переопределить: env `SP9_WORKS_CHAT_ID`)

## Поведение бота в этом чате

- **12:00 МСК (ПН–ПТ):** «Синкуемся?»
- **15:00 МСК (ПН–ПТ):** послеобеденный мем по переписке текущего дня (LLM + fallback-шаблоны)
- **18:00 МСК (ПН–ПТ):** вечерний мем по дневному контексту; в **пятницу** — напутствие на выходные
- **Ответы без @:** шире, чем в других чатах — на рабочие триггеры (`макет`, `синк`, `привет`, `баг`, …) и фоновые слова (`залил`, `готово`, …)
- **С @ или reply:** как везде — реакция в стиле Паши

### Переменные окружения (расписание мемов)

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `SP9_SCHEDULED_MEME_ENABLED` | `1` | `0` — выключить плановые мемы |
| `SP9_AFTERNOON_MEME_HOUR` | `15` | Час послеобеденного мема (МСК) |
| `SP9_AFTERNOON_MEME_MIN` | `0` | Минуты |
| `SP9_EVENING_MEME_HOUR` | `18` | Час вечернего мема (МСК) |
| `SP9_EVENING_MEME_MIN` | `0` | Минуты |

Подробнее про генерацию: [MEME_REPLIES.md](MEME_REPLIES.md).

### Чтобы бот видел сообщения в группе

При **каждом старте** бот проверяет статус в S:P9 works и пишет в лог:

- `✅ SP9 works: бот — administrator` — всё ок, история чата копится для мемов
- `⚠️ SP9 works: бот НЕ администратор` — нужно исправить (см. ниже)

**Исправление (достаточно одного):**

1. @BotFather → **Group Privacy → Turn off**, или
2. Назначить @ag_slashbot **администратором** группы S:P9 works

Без этого в группе работают только команды, @ и reply — **плановые мемы не получат контекст переписки**.

### Как проверить, что всё работает

1. **Старт бота** — в Railway Logs:
   - `✅ SP9 works: бот — administrator` (или Privacy OFF)
   - `🌤️ S:P9 послеобеденный мем … 15:00`
2. **Сообщение без @** в группе → в Logs: `📨 Получено сообщение: …`
3. **После redeploy** → `💾 История чатов загружена` (нужен Volume `/data`; дневная история тоже лежит там)
4. **15:00 / 18:00 МСК** → `🌤️` / `🌆` / `🍻 SP9 scheduled meme` в Logs

Полный чеклист: [OPERATIONS.md](OPERATIONS.md).

## Отправка сообщения вручную (Telegram Bot API)

1) Задай токен в переменную окружения:

```bash
export BOT_TOKEN="вставь_сюда_токен_бота"
```

2) Отправь сообщение:

```bash
curl -sS -X POST "https://api.telegram.org/bot${BOT_TOKEN}/sendMessage" \
  -d "chat_id=-1002413642408" \
  --data-urlencode "text=Синкуемся?"
```

## Проверка названия чата по ID

```bash
curl -sS "https://api.telegram.org/bot${BOT_TOKEN}/getChat?chat_id=-1002413642408"
```

## Полезно

- Не коммить реальный токен в репозиторий.
- Если нужно отправлять часто, можно сделать отдельный скрипт и читать `BOT_TOKEN` из env.
